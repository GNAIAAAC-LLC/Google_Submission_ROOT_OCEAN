## 2. Layer 1 — Media Cluster

### 2.1 Subsystems

- **Script Engine** — content brain (generate, store, translate, schedule scripts)
- **Video Generator** — text → narrated, branded, subtitled video
- **Podcast Generator** — text → normalized, branded audio
- **Blog Generator** — text → structured articles
- **Multi‑Channel Publisher** — YouTube, ViewTube, podcast feeds, blog, email
- **ViewTube** — internal media hub
- **Founder Dashboard** — monitoring, filters, controls
- **AI Sleeping Agent (media scope)** — scheduled, automated content production

### 2.2 Core Flows

1. Script Engine creates/receives scripts.
2. Generators (video, podcast, blog) transform scripts into media.
3. Publisher distributes to configured channels.
4. ViewTube and site surfaces display content.
5. Dashboard tracks jobs, status, and publishing.
6. Sleeping Agent can run the entire flow on a schedule.

## 3. Layer 2 — Intelligence Cluster

### 3.1 Analytics Engine

- Tracks views, listens, reads, watch time, drop‑off, language/topic/channel performance.
- Feeds insights back into Script Engine and Topic Intelligence.

### 3.2 Topic Intelligence Engine

- Consumes analytics + external signals.
- Suggests topics, prompts, calendars, and language mixes.

### 3.3 Multi‑Agent Collaboration Layer

- Agents: Script, Video, Podcast, Blog, Publisher, Analytics, Topic, Sleeping.
- Each agent has a role, responsibilities, and communication patterns.
- Orchestrated via workflows and events.

### 3.4 Intelligence Role

- Turns media into a self‑optimizing, feedback‑driven system.
- Enables autonomous, data‑informed content evolution.
